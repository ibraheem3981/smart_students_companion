Abstract
This project presents the development of a Smart Student Companion, an offline mobile application designed to help students manage their academic activities efficiently. The system integrates features such as timetable scheduling, assignment tracking, GPA calculation, and study timing into a single platform. The application was developed using Flutter and utilizes local storage for offline functionality. The system aims to improve student productivity and organization without requiring internet access. The results show that the application is easy to use, reliable, and effective in supporting student academic planning.

📗 CHAPTER ONE: INTRODUCTION

1.1 Background of Study
Students often face challenges in managing their academic workload. They forget assignments, suffer from poor time management, and rely on too many separate tools to organize their schedules and study sessions.

1.2 Problem Statement
Students struggle to manage academic tasks efficiently due to the lack of a unified, offline solution that caters specifically to their daily needs.

1.3 Aim and Objectives
Aim: To develop a mobile application that helps students manage academic activities offline.

Objectives:
- Design a timetable system for tracking classes
- Build an assignment tracker for deadlines
- Implement a 5.0 scale GPA calculator
- Add a study timer to track weekly study hours

1.4 Significance of Study
- Helps students stay organized and focused
- Works entirely offline without the need for an internet connection
- Combines multiple essential study tools into one app

1.5 Scope of Study
- Only available as a mobile application
- Offline functionality only
- No cloud syncing or online accounts

---

📘 CHAPTER TWO: LITERATURE REVIEW

Existing solutions often fail to meet the specific needs of offline students:
- Google Calendar: Requires an internet connection to sync and function fully.
- Notion: Too complex and requires extensive setup time.
- School portals: Limited features and generally only track official grades rather than personal study time and daily tasks.

Conclusion: Existing systems are either online-dependent or not student-focused. This project provides a simple, dedicated offline alternative.

📙 CHAPTER THREE: METHODOLOGY

3.1 System Design
The application is divided into several modules:
- Dashboard (Overview of classes and assignments)
- Timetable (Daily schedule management)
- Assignment Tracker (Deadline and task management)
- GPA Calculator (Grade tracking)
- Study Timer (Study session and weekly time tracking)

3.2 Tools Used
- Flutter (UI Framework)
- Dart (Programming Language and Logic)
- Local storage (SharedPreferences for offline data persistence)

3.3 Development Approach
An Agile approach was used to build features step-by-step, allowing for continuous testing and iteration of the user interface and local storage functionality.

3.4 System Flow
Open app → View Dashboard → Navigate to specific modules (Timetable, Timer, etc.) → Save data locally on the device.
